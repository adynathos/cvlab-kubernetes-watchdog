CVLab Kubernetes Queue

More information in the CVLab Kubernetes Guide.

We want to ensure that everyone can use at least one GPU. We order the jobs and the position in the queue will decide which jobs will be allowed to run in case we have more requests than available resources.

  • A job using one person's 1st GPU has precedence over any person's 2nd GPU job. A 2nd GPU job is above any 3rd GPU job and so on.
  • Among the jobs of a single user, we order them according to user-set priority (in the label priority) with higher numbers being more important: priority +1 is before priority -1, the default is 0. If priority is equal, the earlier job takes precedence.

If all of the 30 GPUs are occupied, and you want to run your 1st job, you can kill the last job in the queue. In that case please notify the owner.

Please remember to specify your user name and priority in the pod config.

metadata:
  name: username-example-test
  labels:
    user: your-username
    priority: "1" # job with higher priority number takes precedence
